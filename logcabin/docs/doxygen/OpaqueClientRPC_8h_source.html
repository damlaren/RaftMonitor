34  * returns an instance when an RPC is initiated; this can be used to wait for

74  /**
75  * Destructively return the RPC's response.
76  *
77  * This will wait for the RPC response to arrive (if it hasn't already) and
78  * throw an exception if there were any problems. If the reply is received
79  * successfully, this will return the reply, leaving the RPC with an empty
80  * reply buffer.
81  *
82  * This may be used from worker threads only, because OpaqueClientRPC
83  * objects rely on the event loop servicing their ClientSession in order to
84  * make progress.
85  *
86  * \return
87  * The reply, leaving the RPC with an empty reply buffer.
88  *
89  * \throw Error
90  * There was an error executing the RPC. See #getErrorMessage().
91  */
92  Buffer extractReply();
93 
94  /**
95  * If an error has occurred, return a message describing that error.
96  *
97  * All errors indicate that it is unknown whether or not the server
98  * executed the RPC. Unless the RPC was canceled with #cancel(), the
99  * ClientSession has been disconnected and is no longer useful for
100  * initiating new RPCs.
101  *
102  * \return
103  * If an error has occurred, a message describing that error.
104  * Otherwise, an empty string.
105  */
106  std::string getErrorMessage() const;
107 
108  /**
109  * Indicate whether a response or error has been received for
110  * the RPC.
111  * \return
112  * True means the reply is ready or an error has occurred.
113  */
114  bool isReady();
115 
116  /**
117  * Look at the reply buffer.
118  *
119  * \return
120  * If the reply is already available and there were no errors, returns
121  * a pointer to the reply buffer inside this OpaqueClientRPC object.
122  * Otherwise, returns NULL.
123  */
124  Buffer* peekReply();
