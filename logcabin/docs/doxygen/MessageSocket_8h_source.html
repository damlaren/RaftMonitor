30 /**
31  * A MessageSocket is a message-oriented layer on top of a TCP connection.
32  * It sends and receives discrete chunks of data identified by opaque IDs.
33  * Higher layers can use this to build an RPC framework, both on the client
34  * side and on the server side.
35  *
36  * On the wire, this adds a 12-byte header on all messages. The first 8 bytes,
37  * encoded as an integer in big endian, are the message ID. The next 4 bytes,
38  * encoded as an integer in big endian, are the number of bytes of data
39  * following the header. Following the header, the data is sent as an opaque
40  * binary string.
41  *
42  * TODO(ongaro): There are probably two things missing from this header. First,
43  * it should have some mechanism (such as a version field) for changing the
44  * header format later. Second, it should have a checksum, as the
45  * TCP/IP/Ethernet checksums may not be sufficient.
46  */
